System Purpose

The Oyera Auto Service Bay Records Management System is developed as a software engineering project to demonstrate:

  • Functional Requirements: Customer registration, technician registration, record viewing, and navigation
  • Non-Functional Requirements: Usability, responsiveness, maintainability, and accessibility
  • Technical Skills: HTML5 semantic elements, CSS styling, responsive design
  • Project Management: Proper documentation, organized file structure, and version control
